n‑Queens – Backtracking in O(n!) Worst Case

This pseudocode places queens row‑by‑row, backtracking whenever no safe column exists for the current row.

global x[1..n]             // x[r] = column chosen for row r

procedure NQUEENS(k, n):
    for i ← 1 to n do                  // try each column in row k
        if PLACE(k, i) then            // safe square ?
            x[k] ← i                   // place queen
            if k == n then
                OUTPUT_SOLUTION()      // all queens placed
            else
                NQUEENS(k+1, n)        // recurse to next row

procedure PLACE(k, col):               // test (k, col)
    for row ← 1 to k-1 do              // check earlier rows
        if x[row] == col                        // same column
           or abs(x[row] - col) == abs(row - k) // same diagonal
           then return false
    return true

Time grows factorially in the worst case, but the PLACE pruning yields solutions quickly for moderate n (e.g. 8‑Queens). Each recursive return naturally backtracks, erasing the last choice and trying the next column.
